Présentation de l'architecture logicielle

Front : L'application front-end est développée avec React Native et Expo. La gestion des écrans est effectuée à l'aide de composants. Les routes utilisées sont stockées dans des services en fonction du thème de la route (tea/market/authentification).

Back : Le back-end est développé avec NestJS. Nous utilisons la bibliothèque Stripe pour le système de paiement en ligne. L'architecture comprend :

  • L'utilisation d'un script Python pour scrapper les données
  • Un dossier modèle qui regroupe tous les modèles de base de données
  • Une API constituée du service (fonctionnement de la requête), du contrôleur (création de la route) et d'un DTO qui vérifie si les informations envoyées sont sous la bonne forme

BDD : MySQL est utilisé comme système de gestion de base de données.

Présentation de l'environnement technique

Les données lors de la connexion sont stockées dans le local storage (token et id). Nous utilisons l'IDE VS Code aussi bien pour le front que pour le back. Pour une meilleure visualisation de la base de données, nous utilisons DBeaver.

Les fonctionnalités

- Authentification et gestion des utilisateurs.

- Création, mise à jour et suppression de thés dans la bibliothèque.

- Gestion des amis et des interactions sociales.

- Prise en charge des tests unitaires et de bout en bout.

- Support pour le développement, le mode veille et le mode production.

Les lignes de commandes utiles

- Démarrer l'application :

- Développement : `npm run start`

- Mode veille : `npm run start:dev`

- Production : `npm run start:prod`

- Tester l'application :

- Tests unitaires : `npm run test`

- Tests de bout en bout : `npm run test:e2e`

- Couverture des tests : `npm run test:cov`

- Prisma :

- Générer les migrations : `npx prisma migrate dev`

- Appliquer les migrations : `npx prisma migrate deploy`

- Ouvrir le studio Prisma : `npx prisma studio`

- Autres Commandes NestJS :

- Générer un module : `nest g module (nom)`

- Générer un contrôleur : `nest g controller (nom)`

- Générer un service : `nest g service (nom)`
